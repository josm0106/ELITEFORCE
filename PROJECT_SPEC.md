# 엘리트 클로드정리 - 웹 열람 시스템 구축 지침

## 목표
356개 워드(.docx) 파일을 단원별로 정리하여 웹 브라우저에서 열람 가능한 정적 사이트 생성

## 요구사항
1. **단원별 구성**: Vol.1~Vol.10 볼륨별 > 단원별 계층 구조
2. **로딩 최소화**: 선택한 단원만 로드 (lazy loading)
3. **파일 무손상**: 원본 .docx 파일은 변경하지 않음
4. **검색 기능**: 단원명/내용 검색 지원
5. **PDF 제외**: .docx 파일만 대상

## 기술 구현
- **변환**: mammoth.js로 .docx -> HTML 변환 (빌드 타임)
- **검색 인덱스**: 변환 시 제목+본문 텍스트 추출하여 JSON 인덱스 생성
- **프론트엔드**: 단일 HTML 파일 (index.html), 바닐라 JS
- **로딩 전략**: 사이드바에 목차만 표시, 클릭 시 해당 HTML 파일만 fetch
- **원본 다운로드**: 각 단원에서 원본 .docx 다운로드 링크 제공

## 빌드 프로세스
```
node build.js
├── docs/           # 변환된 HTML 파일들
├── search-index.json  # 검색 인덱스
└── index.html      # 메인 뷰어
```

## 사용법
1. `npm install mammoth` (빌드 의존성)
2. `node build.js` (변환 실행)
3. `index.html`을 브라우저에서 열거나 로컬 서버로 서빙
